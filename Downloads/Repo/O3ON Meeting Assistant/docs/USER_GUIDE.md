# User Guide

## Sign in

For local development, enter any email and password. The backend returns a development session.

## Create a meeting

On the dashboard:

1. Enter a meeting title.
2. Enter duration in minutes.
3. Enter participant names separated by commas.
4. Tap `Create meeting`.

## Record

1. Select the meeting.
2. Tap `Record`.
3. Tap `Start recording`.
4. Keep the app visible while recording.
5. Tap `Stop recording` when finished.

## Upload and process

After recording:

1. Review that the capture path is displayed.
2. Tap `Upload and process`.
3. Wait while the app uploads the file and triggers processing.

## Review results

The results screen will:

- show current processing state
- poll the backend while processing is running
- display summary
- display action items
- display the first transcript segments

Possible states:

- `draft`
- `recording`
- `processing`
- `ready`
- `needs_review`
- `failed`

## Expected output

Without live OpenAI credentials:

- transcript and summary will be generated by the stub provider
- output is useful for UI validation, not content quality validation

With live OpenAI credentials:

- transcript quality depends on audio quality and provider availability
- minutes, decisions, and action items are provider-generated
